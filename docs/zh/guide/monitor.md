# 实时监控

实时监控新增媒体项。

::: warning Emby 媒体库硬性要求
本页说的是 ETK 的实时监控。ETK 用户应关闭 Emby 媒体库里的实时监控，让入库刷新由 ETK 统一触发；同时需要开启 Emby 的 NFO 读取，让 Emby 使用 ETK 生成的本地元数据。

如果 Emby 自己监听媒体目录，ETK 的整理、STRM 生成、NFO/海报写入、洗版替换等流程都可能被抢跑。常见表现是 Emby 扫到半成品、重复扫描、全量 STRM 时出现大量入库通知，甚至数据库压力过高。没有开启 NFO 读取时，ETK 生成的标题、简介、分级、海报等本地元数据也可能不会按预期生效。
:::

## 关键配置
![alt text](/public/images/ssjk.png)
- **监控路径**：需要监控的路径列表
- **排除路径**：需要排除的路径列表，必须是监控路径的子路径，命中则跳过刮削仅刷新
- **定时扫描回溯天数**：扫描监控目录任务将只扫描此配置天数内有变化的文件
- **监控扩展名**：仅监控配置了扩展名的文件类型
- **图片语言偏好**：刮削时将优先下载配置的语言图片

## 流程说明

监控到有新文件，程序会立即进行并发刮削、中文化、统一映射美国分级，处理完成会写入覆盖缓存并通知Emby扫描入库

## 常见建议

- 监控目录保持与 Emby 媒体库路径一致。
- 排除“临时下载/缓存”目录，避免误触发。
