# 115 网盘集成

115 网盘是 Emby Toolkit 的核心功能模块，提供完整的媒体文件管理、自动化整理和播放直链服务。

## 功能概述

- **双模式认证**：支持 OAuth 2.0 Token 和 Cookie 两种认证方式
- **智能整理**：基于 TMDb 元数据的自动化文件分类
- **STRM 生成**：自动生成 STRM 文件
- **播放直链**：高效的 302 重定向直链播放，支持本地挂载和远程访问
- **音乐库同步**：独立的音乐库 STRM 生成和附属文件下载
- **历史记忆**：自动记忆手动纠错结果，智能避免重复劳动

## 认证登录

Emby Toolkit 支持两种 115 认证方式，您可以根据使用场景选择：

### 方式一：OAuth 2.0 Token 扫码登录（推荐）

这是官方推荐的认证方式，支持完整的 API 操作权限。

1. 进入 **设置 → 115 网盘** 页面
2. 点击 **获取二维码** 按钮
3. 使用 115 APP 扫描二维码并在手机端确认
4. 系统自动保存 Access Token 和 Refresh Token

**特点**：
- 支持文件管理、目录创建、移动等所有操作
- Token 会自动续期，无需手动处理
- 适用于需要使用智能整理功能的用户

### 方式二：Cookie 认证（轻量级）

如果您只需要播放直链功能，可以使用 Cookie 认证：

1. 进入 **设置 → 115 网盘** 页面
2. 点击 **获取 Cookie 二维码** 按钮
3. 选择 APP 类型（默认支付宝小程序）
4. 扫码确认后自动获取 Cookie

**特点**：
- 仅支持播放直链获取
- 无法使用文件管理功能
- 适合仅需要播放功能的用户

### 方式三：手动输入

对于高级用户，也可以手动粘贴 Cookie：

```bash
# 在 WebUI 的 115 设置页面手动输入 Cookie 字符串
```

### 凭证状态检查

系统会在每次启动时自动检查凭证有效性：

- **Token 有效**：显示绿色状态，可使用所有功能
- **Cookie 有效**：显示黄色状态，仅支持播放
- **凭证无效**：显示红色状态，需要重新登录

## 基础配置

在 **设置 → 115 网盘** 页面配置以下选项：

### 网盘目录设置

| 配置项 | 说明 |
|--------|------|
| 待整理目录 CID | 115 网盘中用于存放待整理文件的目录 ID |
| 未识别目录 CID | 无法自动识别分类的文件存放目录 |
| 媒体库根目录 CID | 分类规则的根目录，用于计算相对路径 |
| 本地 STRM 根目录 | 本地生成 STRM 文件的根目录路径 |

### API 设置

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| API 请求间隔 | 两次 API 请求之间的等待时间（秒） | 0.5 |
| 最大并发线程数 | 整理任务的最大并发数 | 3 |
| 自定义 AppID | 115 应用 ID（高级用户使用） | 系统默认 |

### 功能开关

| 配置项 | 说明 |
|--------|------|
| 启用智能整理 | 是否启用自动整理功能 |
| 下载字幕文件 | 是否自动下载关联的字幕文件 |
| 启用本地清理 | 是否清理本地失效的 STRM 文件 |
| 联动删除网盘 | 是否在删除本地文件时同步删除网盘文件 |

### 文件扩展名

配置允许整理的文件扩展名列表，格式为逗号分隔：

```
mp4,mkv,avi,ts,iso,rmvb,wmv,mov,m2ts,flv,mpg
```

## 智能整理规则

智能整理是 Emby Toolkit 的核心功能，通过规则引擎实现自动化分类。

### 规则配置

在 **115 设置 → 分类规则** 页面创建整理规则：

#### 1. 基本信息

- **规则名称**：规则的显示名称
- **目录名称**：网盘中的目标目录名
- **启用状态**：是否启用该规则

#### 2. 匹配条件

支持以下匹配条件，可多选组合：

| 条件类型 | 说明 |
|----------|------|
| 类型 | 电影 / 剧集 / 全部 |
| 类型 (Genres) | TMDb 电影类型（如动作、喜剧） |
| 国家/地区 | 制作国家/地区 |
| 语言 | 原始语言 |
| 工作室 | 制作公司/发行公司 |
| 关键词 | TMDb 关键词 |
| 分级 | 影片分级（如 PG-13、R） |
| 年份范围 | 上映年份区间 |
| 时长范围 | 影片时长区间（分钟） |
| 最低评分 | TMDb 评分门槛 |
| 演员 | 包含指定演员 |

#### 3. 匹配模式

- **AND 模式**：所有条件都满足才匹配
- **OR 模式**：满足任一条件即匹配

#### 4. 连载状态

针对剧集，可以设置：

- **仅追剧中**：仅整理正在追看的剧集
- **全部**：整理所有剧集

### 重命名配置

在 **115 设置 → 重命名规则** 页面配置：

#### 主标题设置

- **标题语言**：中文 / 原始语言
- **年份格式**：带括号 / 不带年份
- **TMDb 标识**：格式如 `{tmdb=12345}`、`[tmdbid=12345]`

#### 文件命名格式

支持灵活的命名组件组合：

```
title_zh      - 中文标题
title_en      - 原始标题
year          - 年份 (2024)
year_pure     - 纯年份 2024
tmdb_bracket  - {tmdb=ID}
tmdb_square   - [tmdbid=ID]
s_e           - S01E01
season_name_en - Season 01
sep_dash_space - " - "
sep_middot_space - " · "
resolution    - 分辨率 (2160p, 1080p)
codec         - 编码 (HEVC, AVC)
audio         - 音频 (Atmos, TrueHD)
group         - 发布组
```

#### STRM 链接格式

- **标准格式**：`/api/p115/play/{pick_code}`
- **带文件名**：`/api/p115/play/{pick_code}/{filename}`

### 记忆功能

系统会自动记忆手动纠错的结果：

- 当您手动将文件移动到其他分类后，系统会记录该 TMDb ID 对应的分类
- 后续相同媒体将自动使用历史分类，无需重复手动操作
- 分季记忆：剧集的每一季都有独立的记忆

## STRM 生成与同步

STRM 是 Emby/Jellyfin 识别本地媒体文件的特殊格式。

### 全量同步

执行 **全量同步 STRM** 任务：

1. 遍历 115 网盘中的分类目录
2. 获取所有视频文件和字幕文件
3. 生成对应的 STRM 文件到本地目录
4. 自动下载匹配的字幕文件（如果开启）

### 增量同步

在监控模式下，新上传的文件会自动：

1. 识别媒体信息（TMDb ID、类型、标题）
2. 按规则分类到对应目录
3. 生成 STRM 文件
4. 记录整理日志

### 本地清理

开启本地清理功能后，系统会：

1. 对比网盘和本地的文件列表
2. 删除本地已失效的 STRM 文件
3. 清理无 STRM 的空目录

### 挂载模式

如果使用 CD2 等本地挂载方式：

1. 配置 **ETK 服务器地址** 为本地挂载路径
2. STRM 文件内容将为本地路径而非直链
3. 适合原盘玩家，方便神医提取媒体信息

## 播放直链

Emby Toolkit 提供高效的 115 直链播放支持。

### 直链原理

```
客户端请求 → Emby Toolkit → 115 API 获取直链 → 302 重定向到 CDN
```

### 技术特点

- **302 重定向**：最小化服务器带宽消耗
- **内存缓存**：相同文件避免重复请求 115 API
- **UA 模拟**：自动使用客户端真实 UA，避免 403 错误
- **流控保护**：防止触发 115 API 流控
- **CDN 加速**：直链到 115 CDN 节点

### 使用方式

1. 确保配置了有效的 Cookie（播放必需）
2. 默认支持CMS、MH、MP115strm
3. 其他第三方可通过配置正则提取PC支持

### 常见问题

- **播放卡顿**：检查网络到 115 CDN 的连接质量
- **403 错误**：Cookie 可能已失效，需要重新扫码
- **无法播放**：确认 STRM 直链格式正确

## 音乐库同步

独立的音乐库管理模块。

### 配置

1. 设置 **音乐库根目录 CID**
2. 和视频文件共享 STRM 根目录
3. 开启/关闭附属文件下载

### 支持格式

- 音频：mp3, flac, wav, ape, m4a, aac, ogg, wma, alac
- 附属：lrc, jpg, jpeg, png, nfo, txt, cue

### MP联动

1. MP安装webhook插件，请求方式：POST，URL：http://ETK-IP:5257/webhook/emby
2. 存储 & 目录 配置整理到待整理目录，切勿勾选分类、重命名、刮削
3. MP上传完成ETK自动接管归类以及生成STRM

### 影巢联动

1. 影视探索页面配置影巢KEY
2. 选择喜欢的影片点击小心心弹出获取方式选择影巢
3. 选择心仪的资源点一键转存
4. 稍等片刻即可在Emby观赏影片

### 功能

- 自动生成音乐 STRM 文件
- 自动下载歌词和封面图片
- 自动下载 CUE 曲目信息
- 本地失效文件清理

## 任务说明

### 后台任务

| 任务名称 | 说明 |
|----------|------|
| 扫描待整理目录 | 扫描待整理目录，识别媒体并执行整理 |
| 全量同步 STRM | 遍历所有分类目录，生成/更新 STRM 文件 |
| 同步目录树缓存 | 将 115 目录结构同步到本地数据库 |
| 全量同步音乐库 | 同步音乐库 STRM 和附属文件 |
| 清理未识别目录 | 一键清空未识别目录的物理文件 |

### 任务链

系统使用任务链机制：

1. **扫描识别**：多线程并发识别媒体
2. **批量整理**：同一部剧的文件打包处理
3. **STRM 生成**：自动生成对应 STRM
4. **清理空目录**：自动清理整理后的空目录

## 常见问题

### Q: Token 和 Cookie 有什么区别？

**Token**：完整的 API 权限，可进行文件管理、整理操作
**Cookie**：仅支持播放直链获取，无法使用整理功能

### Q: 整理失败怎么办？

1. 检查 115 API 是否触发流控（等待后重试）
2. 查看任务日志了解具体错误
3. 手动将文件移动到正确目录，系统会记忆结果

### Q: 如何查看整理历史？

在 **整理 → 整理记录** 页面查看：

- 整理成功/失败记录
- TMDb 匹配信息
- 目标分类路径
- 支持手动纠错

### Q: 为什么不自动下载字幕？

1. 确认已开启「下载字幕文件」开关
2. 检查文件扩展名是否在允许列表
3. 115 网盘中必须存在字幕文件
