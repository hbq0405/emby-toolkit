# 共享资源中心

共享资源中心用于让多台 ETK 客户端共享自己已经入库的媒体资源。简单说：别人有的资源，你可以直接秒传或转存；你提供的资源被别人使用，也会获得贡献点。

它适合这些场景：

- 自己缺某部电影或某一季剧，想从共享池快速补齐。
- 多个朋友各自维护媒体库，希望互相借力。
- 想把高质量版本、杜比版本、HDR/SDR 版本集中到一个资源池里，方便挑选。
- 希望通过一致性校验避免剧集版本混杂。

## 先理解几个概念

### 中心资源库

中心资源库是公共资源池的浏览入口。这里展示已经被中心端收录的电影、剧集和季包。

- 电影按 **TMDb ID** 聚合成一张卡片。
- 同一电影下不同清晰度、编码、音轨、HDR 等版本会放在详情页资源列表里。
- 同一电影、同一 SHA1 的重复来源会聚合成一个资源，不会重复占位。
- 剧集会按剧名聚合，详情页再按季查看资源。

### 秒传

秒传适合已经有资源签名能力的资源。ETK 会向中心申请许可，再通过 115 秒传能力完成入库。

常见按钮文案是 **秒传**。

### 转存

如果资源有可用的 115 分享，ETK 会展示 **转存**。转存不依赖签名客户端在线，但依赖分享链接仍然有效。

### 贡献点

贡献点用于限制公共资源的滥用。

- 使用别人资源时会消耗贡献点或 Pro 额度。
- 你的分享资源被别人成功转存时，会获得贡献点。
- 求共享退款、Pro 额度抵扣等也会出现在贡献点明细里。

## 开启共享资源

进入 **Web 控制台 → 共享资源**。

通常需要完成这些准备：

1. 已配置并可正常使用 115 网盘。
2. 本地媒体库已经完成入库，媒体有 TMDb 信息。
3. 后台任务能正常运行。
4. 共享资源中心地址、设备身份等配置已保存。

开启后，ETK 会把本地可共享资源登记到中心端。登记的只是资源索引和必要元数据，不会把视频文件上传到中心服务器。

## 使用中心资源库

进入 **共享资源 → 中心资源库**。

你可以：

- 搜索电影或剧名。
- 按电影、剧集筛选。
- 查看资源标签，如 4K、HDR、杜比、国语、中字、纯净版、短剧等。
- 打开详情页选择具体版本。
- 对可用资源执行秒传或转存。

首屏只展示卡片需要的信息。演员、简介、资源列表等内容会在详情页加载。

## 电影资源怎么看

电影卡片按 TMDb ID 聚合。也就是说同一部电影只应该有一张卡。

进入详情页后，才会看到不同资源版本，例如：

- 1080p / 2160p
- AVC / HEVC
- HDR / SDR / Dolby Vision
- 不同音轨和字幕
- 不同文件体积

同一个 SHA1 会聚合成一个资源，避免同一文件重复显示。

## 剧集资源怎么看

剧集卡片按剧名聚合。

详情页默认显示一个季，切换季标签后才加载对应季资源。这样首屏更快，也避免一次加载所有季导致页面卡顿。

常见状态：

- **连载中**：正式季还在更新，或中心端还没有确认完结。
- **已完结**：正式季已经完结，但不一定代表版本一致性已经通过。
- **一致版**：中心端确认该季集数完整且版本一致，适合整季收藏。

特别篇不参与正式季完结判断。

## 纯净版、短剧和动漫

中心端会根据上传的媒体元数据和视频物理时长判断：

- **短剧**：集时长明显较短的剧集。
- **纯净版**：可能去掉片头片尾或广告的版本。
- **动漫**：动漫不参与短剧、纯净版判断。

如果你在设置里开启了“不秒传短剧”或“不秒传纯净版”，ETK 会在自动秒传时拦截对应资源。

## 求共享

如果中心资源库里没有你想要的资源，可以使用 **求共享**。

求共享会消耗贡献点作为悬赏。其他设备如果提供了符合条件的资源，中心端会自动匹配并通知。

适合填写的条件包括：

- 电影或剧集名称
- 清晰度
- 编码
- HDR / 杜比
- 音轨和字幕偏好
- 体积范围

条件越多，匹配越严格，也越可能等得更久。

## 我的共享源

**我的共享源** 显示你这台设备登记到中心端的资源。

你可以检查：

- 哪些资源已经登记。
- 分享状态是否有效。
- 本地资源是否已经离线。
- 是否需要执行维护任务。

如果你已经从 115 删除了分享，可以执行共享维护或同步任务，让中心端清理失效记录。

## 共享维护任务

共享维护会做这些事：

- 补齐中心端缺失的海报、简介、演员、集时长等元数据。
- 同步有效分享状态。
- 清理本地已经不存在的共享源。
- 修复中心端和本地之间的脏数据。
- 维护秒传签名能力。

当你发现中心资源库展示异常、分享状态不一致、资源标签缺失时，可以先执行一次维护任务。

## 成人资源

成人资源适合独享，不适合进入公共共享池。

如果媒体入库时被识别为成人内容，ETK 会将分级归一为 `XXX`，共享登记前会直接跳过，不上传 RAW，也不登记到中心端。

## 常见问题

### 为什么同一部电影只显示一张卡？

这是正常的。中心资源库首屏按 TMDb ID 聚合，同一部电影只显示一张卡。不同版本请进入详情页查看。

### 为什么详情页有多个资源？

不同 SHA1 代表不同物理文件，可能是不同分辨率、编码、音轨、字幕或体积版本。详情页会保留这些版本供你选择。

### 为什么两个资源看起来完全一样？

有些资源虽然参数相似，但文件 SHA1 不同。它们会作为不同资源展示。你可以根据体积、标签、热度和来源选择。

### 为什么秒传一直排队？

通常是中心端正在等待可用签名设备，或者同一资源短时间内已经签名过，中心端会避免高频重复签名。

不同资源之间不会互相强制排队。

### 为什么只能转存，不能秒传？

说明这个资源当前有可用 115 分享，但没有可用签名能力。可以直接使用转存。

### 为什么资源消失了？

可能原因：

- 提供者删除了本地资源。
- 115 分享失效。
- 资源 RAW 信息缺失。
- 中心维护任务清理了无效数据。

### 贡献点明细里的标题是什么意思？

贡献点明细会显示扣点或加点原因，例如：

- 秒传资源消耗
- 转存 115 分享资源
- 贡献资源奖励
- 求共享退款
- Pro 额度抵扣

标题会尽量显示对应影片或剧名，方便你知道这笔流水来自哪个资源。

## 使用建议

- 先确认本地 115 和 Emby 入库正常，再开启共享。
- 批量刷新剧集时不要反复创建分享。
- 发现资源状态不对，先执行共享维护。
- 不要反复尝试分享明显违规的资源。
- 对收藏党，优先选择一致版季包或热度更高的资源。
