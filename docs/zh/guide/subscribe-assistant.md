# 订阅助手增强版

订阅助手增强版移植自爱神的 MoviePilot 订阅助手增强版插件，用于在 ETK 中全流程接管 MoviePilot 订阅。

旧 ETK 策略更像是“判断一次，然后向 MoviePilot 发一次指令”。订阅发出去以后，后续是否误完结、是否被 TMDb 集数变化影响、下载是否卡住、订阅是否丢失，ETK 很难继续维护。

订阅助手增强版改成持续接管：ETK 会记录订阅为什么待定、为什么暂停、什么时候收口、当时的完成快照，并通过周期维护继续检查。

## 入口

进入：

```text
智能追剧列表 -> 策略配置 -> 订阅助手增强版
```

打开“增强订阅助手接管 MoviePilot”后，MoviePilot 订阅状态由订阅助手统一同步。

## 总体行为

ETK 追剧刷新后，会把最终状态交给订阅助手处理：

| ETK 状态 | MoviePilot 订阅行为 |
| --- | --- |
| 追剧中 | 订阅保持运行 |
| 已暂停 | 订阅暂停 |
| 待定中 | 订阅设为待定，并写入虚标集数 |
| 已完结 | 保存完成快照，进入完结守卫与后续维护 |

旧的自动待定、自动暂停、完结洗版、MP 同步、版本锁定开关已经合并进订阅助手配置。增强同步失败时不会再回退旧策略，避免两套逻辑互相覆盖。

## 推荐配置

普通追剧建议：

| 配置项 | 推荐值 |
| --- | --- |
| 接管开关 | 开启 |
| 守卫模式 | 平衡 |
| 季冷却期 | 14 天 |
| 启用波动判断 | 开启 |
| 增强待定 | 开启 |
| 待定虚标集数 | 99 |
| 增强暂停 | 开启 |
| 开播前暂停 | 14 天 |
| 播出间隔暂停 | 30 天 |
| 自动洗版范围 | 剧集 |
| 自动纠错 | 开启 |
| 下载巡检 | 先关闭，确认环境稳定后再开启 |

如果经常追新番、国产综艺、短剧，建议把“守卫模式”改成“严格”，并打开“待定参考波动”。

## 完结守卫

完结守卫用于防止 TMDb 数据不稳定导致假完结。

常见情况：

- TMDb 当天显示某季只有 3 集，几天后变成 8 集。
- 本季 finale 标记提前出现，但后面仍有追加集。
- 新番、综艺、短剧总集数经常变动。

配置说明：

| 配置项 | 作用 |
| --- | --- |
| 守卫模式 | 控制完结判断保守程度 |
| 季冷却期 | 完结后等待一段时间再认为稳定 |
| 集数波动窗口 | 观察 TMDb 集数变化的时间范围 |
| 启用波动判断 | 集数近期变化时，不轻易收口 |

守卫模式：

| 模式 | 说明 |
| --- | --- |
| 平衡 | 默认推荐，低置信完结会观察 |
| 严格 | 更保守，适合新番、综艺、短剧 |
| 宽松 | 更积极收口 |
| 关闭 | 不做完结守卫 |

## 待定与暂停

待定用于处理刚开播、集数少、信息不稳定的剧。暂停用于处理下一集离现在很远的剧。

| 配置项 | 作用 |
| --- | --- |
| 增强待定 | 新剧、低集数、数据不稳定时进入待定 |
| 参考波动 | 待定判断时参考 TMDb 集数近期变化 |
| 待定虚标集数 | 写给 MoviePilot 的目标集数，防止提前完成 |
| 增强暂停 | 根据播出日期暂停 MoviePilot 订阅 |
| 开播前暂停 | 距离开播还很远时暂停 |
| 播出间隔暂停 | 下一集距离现在超过阈值时暂停 |

“参考波动”和“启用波动判断”的区别：

- 启用波动判断：主要服务于完结守卫，防止假完结。
- 参考波动：主要服务于待定判断，防止刚开播集数少被误判。

## 洗版编排与版本锁定

洗版和锁版是两件事。

- 洗版负责判断新文件要不要替换旧文件。
- 锁版负责在合适的时候把 MoviePilot 订阅限制到某一类版本，避免后面继续下载不想要的版本。

| 自动洗版范围 | 行为 |
| --- | --- |
| 关闭 | 不带洗版参数，也不主动做版本锁定 |
| 剧集 | 剧集订阅带洗版倾向，普通追剧推荐 |
| 剧集分集 | 偏分集洗版 |
| 全部 | 更激进，相关订阅都带洗版倾向 |

`分集转全集` 是完结后的额外动作：追更阶段可以先按分集洗版追更，完结后再把 MoviePilot 订阅改成全集洗版，用完结包替换零散分集。

如果打开 `一致性校验`，分集转全集会先检查本地这一季是否已经版本一致：

- 一致性通过：不再提交全集洗版订阅，避免已经整齐的本地文件又去下载完结包。
- 一致性不通过：标记这一季 `active_washing`，提交全集洗版订阅；后续入库继续校验，直到一致性通过后清理标记。

如果关闭 `一致性校验`，只要订阅助手判断该季完成，并且开启了 `分集转全集`，就会直接提交全集洗版订阅。

有新分集入库后，ETK 会从文件名提取版本特征，例如：

- `WEB-DL`
- `2160p`
- `HDR`
- `DDP`
- 发布组

### 锁版模式

锁版模式保留旧版习惯，只提供两个常用选项：

| 锁版模式 | 行为 |
| --- | --- |
| 关闭 | 不主动改 MoviePilot 订阅包含规则 |
| 最佳版本 | 第一集入库后先等待优先级 1；等不到时按“降级间隔”逐步接受优先级 2、3 |
| 任意版本 | 第一集入库后，按当前入库版本直接锁定 MoviePilot 订阅 |

最佳版本适合强迫症追剧。它不是永远死等理论最佳，而是从第一集真正入库时间开始计时：

```text
0 个降级间隔内：只接受优先级 1
1 个降级间隔后：接受优先级 1 或 2
2 个降级间隔后：接受优先级 1、2 或 3
```

例如降级间隔设为 6 小时：

```text
第 0-6 小时：优先级 1 入库才锁版
第 6-12 小时：优先级 2 也可以锁版
第 12 小时后：优先级 3 也可以锁版
```

任意版本适合只想尽快稳定的场景。谁先入库，就按谁的版本特征锁定后续订阅。

锁版起算时间使用媒体资产里的 `date_added_to_library`，不是 `last_updated_at`。刷新元数据、维护任务、纠错任务不会重置锁版等待时间。

锁版会尝试从入库文件名提取版本特征，然后更新 MoviePilot 订阅的包含规则，让后续订阅更偏向同一版本。洗版本身不会写 MoviePilot 包含规则，所以不会和锁版抢同一个配置。

## 自动纠错

自动纠错处理的是“当时判断正确，但后来事实变了”的情况。

典型例子：

1. 某剧 S1 当时 TMDb 显示 8 集。
2. 订阅助手判断可收口，保存完成快照。
3. 过几天发现 MoviePilot 订阅丢失，或者 MP 订阅总集数低于快照。
4. 自动纠错会按快照恢复订阅、修正 MP 订阅总集数，并可触发补搜。

自动纠错不会覆盖豆瓣或手动锁定的集数。

优先级是：

```text
豆瓣/手动锁定集数 > 自动纠错快照 > TMDb 当前集数
```

也就是说，如果豆瓣辅助把某季锁定为 12 集，自动纠错不会因为快照或 TMDb 变化把它改成 13 集。锁定季只会记录日志并跳过 MP 恢复动作。

配置说明：

| 配置项 | 作用 |
| --- | --- |
| 启用纠错 | 开启完成快照检查 |
| 纠错间隔 | 多久检查一次快照 |
| 快照保留 | 快照保留多少天 |

## 豆瓣辅助修正集数

豆瓣辅助修正集数是最高优先级的本地集数锁定。

开启后，ETK 会在追剧刷新时尝试从豆瓣获取最新季总集数。获取成功后会写入本地 Season 总集数，并标记为锁定。

锁定后：

- 后续追剧判断优先使用锁定集数。
- TMDb 多出来的集数会被过滤。
- 自动纠错不会覆盖该锁定值。
- 手动修正集数也属于同级高优先级锁定。

这个功能适合 TMDb 集数经常滞后或错误的国产剧、综艺。

## 下载监控

下载监控用于处理 MoviePilot 下载任务长期无进度、任务被删除、需要补搜等情况。

| 配置项 | 作用 |
| --- | --- |
| 下载巡检 | 周期检查下载任务状态 |
| 手动删种监听 | 下载任务消失时清理订阅助手状态 |
| Tracker 响应监听 | 识别 Tracker 异常关键字 |
| 超时窗口 | 多久无有效进度算停滞 |
| 进度阈值 | 窗口内至少应增长多少进度 |
| 重试次数 | 连续停滞多少次后进入保护 |
| 删种后补搜 | 删除异常任务后触发订阅搜索 |
| 删除记录保留 | 避免近期删除资源反复命中 |

建议先保持关闭。确认 MoviePilot API、下载器和 ETK 周期任务都稳定后，再开启下载巡检。

## 订阅清理

订阅清理用于控制订阅完成、恢复、删除、超时等场景下是否清理历史记录。

当前配置已进入订阅助手统一配置模型，后续会继续接入更细的 MoviePilot 历史清理动作。

## 识别增强

识别增强用于判断资源候选是否高风险。

目标是拦截这些情况：

- 资源季集范围和订阅目标不一致。
- 完结包集数明显不匹配。
- 候选资源可能导致错季、错集、误洗版。

配置说明：

| 配置项 | 作用 |
| --- | --- |
| 候选准入 | 开启增强识别 |
| 仅记录 | 发现风险只记日志 |
| 拦截高风险 | 只拦明显不靠谱的资源 |
| 严格拦截 | 更保守，风险候选更容易被拒 |

需要注意：ETK 不是 MoviePilot 插件运行环境，没有 MoviePilot 原生的 ResourceSelection 事件总线。当前版本已经保留策略框架，事件级候选拦截还需要后续把 MoviePilot 的候选或下载事件接入 ETK。

## 周期维护任务

订阅助手维护任务会执行：

- 清理过期快照。
- 检查完成快照。
- 清理过期删除记录。
- 在开启下载巡检时检查下载任务。

正常情况下不需要手动运行。修改策略后，下一轮追剧刷新或订阅助手维护任务会逐步生效。

## 常见问题

### 和旧 ETK 订阅策略有什么区别？

旧策略是一次性下发指令。订阅助手增强版是持续接管 MoviePilot 订阅生命周期。

### 快照是不是截图？

不是。快照是订阅完成时的状态记录，包括 TMDb ID、季号、目标集数、订阅 ID 和当时的订阅对象。

### 自动纠错会不会和豆瓣辅助冲突？

不会。豆瓣或手动锁定集数优先级最高。锁定季会跳过自动纠错动作。

### 版本锁定在哪里？

在“洗版编排 -> 自动洗版范围”。选择“剧集”或“剧集分集”后，新分集入库时会尝试按文件版本特征锁定订阅。
